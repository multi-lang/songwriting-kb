# Analysis Reports

This directory stores critique reports, scoring records, and analysis outputs generated during the songwriting pipeline.

## File Naming Convention

| Source | Report Name |
|---|---|
| `songs/album_act2/08_First_Blood.md` | `08_First_Blood_critique.md` |
| `songs/experimental/The_Wound_Speaks.md` | `The_Wound_Speaks_critique.md` |
| `songs/album_act3/18_Edge_of_the_Rift.md` | `18_Edge_of_the_Rift_critique.md` |

## Report Types

| Type | Suffix | Generated By | Contains |
|---|---|---|---|
| **Critique** | `_critique.md` | Critic agent (SOP 02) | 12-category scores, strongest line, flagged issues, recommendations |
| **Continuity** | `_continuity.md` | Album-Continuity agent | Rule check results, palette compliance, motif verification |
| **Optimization** | `_suno_optimization.md` | Suno-Optimizer agent | Changes made, char counts, slider recommendations |
| **Comparison** | `_comparison.md` | Manual/pipeline | Self-critique vs independent critique gap analysis |
| **Revision Log** | `_revision_log.md` | Songwriter (revision stage) | What was changed between versions and why |

## Report Format

All reports follow this structure:

```markdown
# [Song Title] — [Report Type]

**Date:** YYYY-MM-DD
**Version analyzed:** v[X] (first draft / revised / final)
**Analyzer:** [Critic agent / Self-critique / Independent]

---

[Report content per type-specific format]

---

**Next action:** [What happens next based on this report]
```

## Retention

- Keep ALL reports — they document the creative decision history
- Reports are never deleted, only superseded by newer versions
- Use reports to identify patterns (recurring weaknesses, improving scores)

## Using Reports for Improvement

After 5+ songs have been critiqued, review reports for patterns:
- Which category consistently scores lowest? (Focus practice there)
- Which issues recur? (May need a steering rule update)
- What's the average self-critique vs independent gap? (Calibration)
- Which song types score highest? (Double down on strengths)
