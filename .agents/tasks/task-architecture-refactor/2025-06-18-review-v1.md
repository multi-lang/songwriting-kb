# Methodology as single source of truth

This change extracts methodology content from Kiro agents, SOPs, and skills into five canonical files under `core/methodology/`, then rewires agents and one skill to be thin wrappers that reference those files via `#[[file:]]`. It also adds a format-validation script, an experiment-logging structure for Suno behavior claims, a version-status table, and a "Using Without Kiro" section in the README. The refactor eliminates multi-file drift by collapsing 3-5 duplicated sources per workflow into one.

**Watch for:** The rewiring is incomplete — the `character-voice` skill still points to the SOP instead of the new methodology file (confirmed). The SOPs directory survives untouched, creating a parallel source of truth that can drift from the canonical files (likely). The validation script's "genre first" check is permissive enough to pass any short string as a genre (confirmed).

## High-level view

The core architectural move is five self-contained methodology files, each declaring itself the single source of truth, each readable without Kiro tooling. Agents become persona + file-references. This solves the drift problem for the four agents that were rewired.

The rewiring is unfinished. The `character-voice/SKILL.md` still references `.kiro/sops/06-character-voice-design.md` rather than `core/methodology/character-voice.md`. More significantly, the eight SOP files in `.kiro/sops/` remain unchanged and now contain methodology that may diverge from the canonical files as either side evolves. The architecture plan's intent was "everything else REFERENCES it" — the SOPs currently don't.

The validation script's genre-first check cannot distinguish an actual genre from an arbitrary short string, so it passes trivially. The experiment structure and version-status table address the Suno-version-dating problem with a clear promotion rule (3+ renders across different songs before a claim becomes guidance).

## Incomplete rewiring of the character-voice skill

The `song-critique/SKILL.md` was rewired to reference `core/methodology/critique.md`. The four agent files (`critic.md`, `songwriter.md`, `suno-optimizer.md`, `album-continuity.md`) were all rewired. But `.kiro/skills/character-voice/SKILL.md` still contains `#[[file:.kiro/sops/06-character-voice-design.md]]` as its primary reference. The new `core/methodology/character-voice.md` exists (300 lines, complete 11-step procedure) but nothing in the `.kiro/` tree points to it. A user activating this skill gets the old SOP content, not the canonical methodology.

## The orphaned SOP layer

The architecture plan explicitly states the goal: "Everything else REFERENCES [core/methodology/] rather than duplicating it." The agents achieved this. The SOPs did not. The eight files in `.kiro/sops/` each contain full procedural content that now exists in parallel with `core/methodology/`. If someone reads `.kiro/sops/02-critiquing-a-song.md` directly (or if a steering file loads it), they get the old version with no pointer to the canonical source.

The drift problem the refactor was meant to solve still exists for anyone who reads or references the SOPs directly. The contributing SOP (`08-contributing.md`) tells contributors to edit skills and agents but doesn't mention `core/methodology/` as the place where actual content changes should land.

## Validation script genre-first gap

`check_genre_first` checks only that the first comma-separated element doesn't match a small set of technical patterns (`tempo`, `key`, `bpm`, `vocal range`) and is under 100 characters. A Style Prompt starting with "ethereal, reverb-heavy, female soprano, dark ambient" passes despite "ethereal" not being a genre — it's a descriptor. The methodology states genre must be first; the validator can't enforce that distinction without a genre vocabulary. This makes one of the seven checks effectively cosmetic.
